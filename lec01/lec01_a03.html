네트워킹 돌아가기

소켓이라는 장치는 메시지를 보내고 받는 장치입니다. 소켓이란 데이터를 연결하는 중간 연결 고리입니다. 자바에서 소켓은 java.net.Socket에 정의되어 있으며, 이 객체를 생성해 사용하면 됩니다.

ServerSocket은 데이터를 받는곳에서만 사용하는 추가적인 장치입니다. 사용자의 연결이 들어오면 ServerSocket을 통해 이를 accept하여 연결된 Socket을 구합니다. ServerSocket은 서버 내에서 외부 다른 소켓들이 연결되기를 기다리는 역할을 합니다. 이런 이유 때문에 흔히 '리스너(Listener)'라고 부르기도 합니다. ServerSocket을 만들 때에는 포트 번호만 있으면 됩니다. ServerSocket의 accept 메서드는 소켓 연결을 기다립니다.

Socket을 연결해서 통신하는 구조를 천천히 살펴보면 다음과 같습니다.

  1. 서버 쪽에서는 ServerSocket 객체를 생성하고 연결을 기다립니다.
  2. 외부에서 접속하는 사람은 Socket 객체를 만들어서 서버에 접속을 시도합니다.
  3. 서버에서는 accept() 메서드가 실행되면서 실제로 클라이언트와 연결된 Socket 객체가 만들어집니다.

DataOutputStream, DataInputSteam을 이용하여 문자열 데이터를 주고 받으면 InputStream과 OutputStream의 read(), write()메서드를 사용하는 것보다 편합니다. byte[]를 이용하지 않으므로 문자열의 길이 같은 정보를 신경쓰지 않아도 되고, writeUTF()와 같은 메서드도 사용할 수 있기 때문입니다. DataOutputStream과 DataInputSteam을 사용하기 위해서는 결국 InputStream과 OutputStream을 생성자에 넣어주어야 하는데, 이러한 식으로 클래스를 가공하여 사용하는 스킬이 있으면, 해야할 일이 줄어듭니다.
