Java

Intro 돌아가기

Java에 대한 개념 정리에 앞서 Java에 대한 소개를 하려고 합니다. Java가 현재에 오기까지는 어떤 일들이 있었는지 한번 되짚어보려고 합니다.

Java는 제임스 고슬링(James Gosling)에 의해 개발된 언어로 초기에는 가전에품에 탑재될 프로그램을 만들기 위한 목적으로 설계되었습니다. 하지만 당시(1990년대)에는 가전제품을 판매하는 측면에서 이러한 프로그램은 너무 시대를 앞서나가는 것이었고, 따라서 외면당했습니다.

Java가 본격적으로 주목을 받게된 것은 인터넷의 보편화와 함께입니다. Java라는것 자체가 다양한 가전제품에서 네트워킹 할 수 있는 작은 시스템을 목적으로 만들어졌기 떄문에, 특정 OS에 종속적이지 않았고, 따라서 다양한 기기에 탑재될 수 있었습니다. 즉, Java는 OS에 종속적이지 않은 언어입니다. Java의 실행은 가상 머신(JVM)에 의해 이루어집니다.

자바는 운영체제의 영향을 받지않고, 각기 다른 환경에서 같은 코드로 GUI화면을 보여줄 수 있는 AWT라는 기술을 고안해냈습니다. 개발자는 Java로 코드만 작성해주면, JVM이 알아서 각 운영체제에 알맞는 화면을 만들어주는 것입니다. 이는 처음 목적이었던 소형 가전이 아닌 웹에서 더 주목을 받게되는데, 그 이유는 인터넷 화면 때문입니다. 당시에는 화려한 웹페이지 기술이 없었는데 Java의 애플릿(Applet)이 웹싸이트 화면을 구성하는데 알맞다는것을 사람들이 알게 되면서 Java는 인터넷에서 주목을 받게 됩니다.

애플릿이 인기를 끌면서 자바는 원래의 목적인 가전이 아닌 인터넷에서 사용하는 언어로 발전하기 시작합니다. 따라서 현재의 Java는 Web을 Target으로 한 범용언어라고 할 수 있습니다. Java를 개발한 Sun사에서는 일반적인 Java 프로그램인 Java SE(Standard Edition), 기업 환경에 맞는 Java EE(Enterprise Edition)을 1999년에 발표합니다. 여기서 자바 EE는 굉장한 성공을 거두게 됩니다.

Computing Paradigm의 변천 돌아가기

본격적인 Java 학습에 앞서, 먼저 컴퓨터 기술이 어떻게 발전해왔나를 살펴보고자 합니다. 여러분 생각에는 "개발자 편의성" 이라는 단어가 어떻게 다가오시나요? 개발에 있어서 개발자 편의성이 고려되어야 할까요? 이 질문에 대한 답은 각기 다를 수 있지만 확실한 것은 현재 기술은 개발자 편의성이 증대되는 방향으로 나아가고 있습니다. 사실 이것은 당연한 이치입니다. 매일 똑같은 반복적인 일처리를 해가며 야근을 할 것입니까? 아니면 그런것은 효율적으로 처리한 후 다른 일을 할 것입니까? 우리는 개발할 때, 혹은 기술을 배울때, '이 기술이 내가 편하기 위한 기술인가?'에 대해 한번쯤 고려해볼 필요가 있습니다.

컴퓨터 기술도 계속 변해왔습니다.포트란이나 코볼같은 절차적인 언어가 중심적인 역할을 하던 때도 있었죠. 90년대에 Java가 출현한 이후, 2005년 부터 떠오르기 시작하는 각종 Framework를 그냥써도 되는가?하는 질문에 대한 답은 무조건 Yes!!라고 할 수는 없겠지만, 기술은 늘 개발자 편의성의 중대되는 방향으로 발전해왔고, 발전해가고 있습니다. 그리고 많은 기술 변화를 거쳐, 현재 우리는 기술들이 공존하는 빅 데이터 시대에 있습니다.

아무튼 우리가 배우고자 하는 언어는 Java이니 Java가 개발자 편의성을 얼마나 고려해주는지 한 번 따져보도록 합시다. 자바의 장점은 다음과 같습니다.

  • 플랫폼 독립성
  • 객체지향과 재사용성
  • Garbage COllection에 의한 메모리 관리
  • Multi-Thread 지원
  • Try~Catch문에 의한 손쉬운 예외처리
  • 강력한 보안 매커니즘
