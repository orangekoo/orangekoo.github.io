Java

String 클래스 돌아가기

String 클래스의 주요 메서드는 다음과 같습니다.

  • concat(추가할 문자열): 문자열을 연결할 때
  • substring(자르기 시작할 위치, 자르고 싶은 길이): 문자열을 자르고 싶을 때
  • length(): 문자열의 길이를 리턴
  • charAt(위치): 문자열 중에서 특정한 위치의 글자를 찾고 싶을 때
  • indexOf(찾고자 하는 문자열): 자신의 원하는 특정 문자열의 위치
  • equals(비교대상 문자열): 다른 문자열과의 내용의 비교
  • trim(): 문자열 뒤에 있는 공백을 제거하고자 할 때
  • split(토근): 문자열을 특정 토큰으로 분리

StringBuffer, StringBuilder 돌아가기

String은 속도 면에서 너무나 느리기 때문에 가능하면 다른 클래스를 활용하는 것이 좋습니다. String은 String Literal Pool에 들어가는 불변의 객체이기 때문에 메모리 내에서 작업이 많이 이루어 지기 때문입니다. 여기서 StringBuffer는 동기화 이슈를 지원하기위한(?) key를 지원하고, StringBuilder는 이 기능이 없어서 StringBuilder가 더 빠릅니다. 사용할 때에는 String과 다르게 +가 아닌 append() 메서드를 이용합니다.

Calendar 돌아가기

날짜를 처리할 때는 Celendar 계열 클래스를 사용합니다. java.util.Calendar 클래스 혹은 java.util.GregorianCalendar 클래스를 가장 많이 사용합니다. 이에 대한 사용법은 Java API 문서에 자세히 나와있습니다.

Date 클래스 돌아가기

Date는 Calender와 long의 중간 고리 역할을 합니다. 과거에는 Date가 날짜 처리의 주요 클래스 였으나, JDK1.1 부터 날짜 변경선 등의 문제와 같은 현실적인 지역별 문제가 많이 고려되여 사장되었습니다.

Scanner 클래스 돌아가기

Scanner는 사용자로부터 입력을 받거나 파일에서 데이터를 읽어낼 수 있는 클래스 입니다. 참고로 Scanner는 Java 5부터 지원됩니다.

Wrapper 클래스 돌아가기

Wrapper는 기본 자료형을 객체 자료형으로 만듭니다. Integer, Double, Byte, Long, Float, Character, Boolean, Short 등이 있습니다. 모든 Wrapper 클래스는 그 자체가 변경 불가능한 객체만 생산 할 수 있으므로, 연산을 한다고 해도 기 결과가 원래 객체를 덮어 씌우는 것이 아니라 새로운 객체로 만들어져 참조하게 됩니다. Java 5부터 오토 박싱과 오토 언박싱 기능이 제공되어 변수 측면에서의 제약 사항이 없어졌는데, 코드의 퍼포먼스를 고려한다면 지양하는 것이 좋습니다.

Formatter 클래스 돌아가기

Formatter는 날짜나, 숫자, 데이터의 포매팅에 대한 통일된 방법을 제공합니다. printf라는 메서드도 있지만 이는 보안에 좋지 않기 떄문에, Formatter를 사용하는 것이 좋습니다. Formatter 클래스는 데이터를 String 타입의 반환 값으로 사용할 수 있는 장치를 제공합니다.
